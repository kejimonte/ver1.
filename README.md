## サービス概要
日々の生活の中で感じた「ありがとう」を記録するシンプルな日記アプリです。
小さな感謝を可視化し、振り返ることで心の安定や自己肯定感を育てます。
他人に見せるためではなく、自分の人生を支えるための記録を目的としています。

## このサービスへの思い・作りたい理由
私はこれまで、
「何のために生きているのか」「自分は幸せなのか」といった問いを
無意識に考え続けながら生活してきました。

日中は仕事や人との会話で気が紛れる一方、
夜になると孤独感や虚無感が強くなり、心が沈むことが多くありました。

そんな中で振り返ると、
自分を救ってくれたのは大きな成功や成果ではなく、

誰かと笑って話した時間

支えられた一言

何気ない優しさ

といった、日常の中にある小さな出来事でした。

しかし、忙しい日々の中ではそれらはすぐに流れてしまい、
意識しなければ「なかったこと」になってしまいます。

そこで、
自分が確かに支えられて生きている証を残せる場所として
「ありがとう日記」を作りたいと考えました。

このサービスは、
誰かに評価されるためのSNSではなく、
自分の人生を肯定するための記録アプリです。

## ユーザー層について
想定ユーザー

20代〜30代の社会人

仕事や人間関係で疲れを感じている人

夜になると孤独感や虚無感を感じやすい人

この層を対象にした理由

自分自身がこの層に属しており、
同じように「頑張っているのに心が満たされない」
と感じている人は多いと感じたからです。

また、
派手なSNSよりも「静かに自分と向き合える場所」を求めている層に
価値を提供できると考えました。

## サービスの利用イメージ
ユーザーはその日に感じた「ありがとう」を短い文章で記録します。
過去の投稿を振り返ることで、
自分がどれだけ人や出来事に支えられてきたかを実感できます。

気分が落ち込んだ時に見返すことで、
「自分は一人ではなかった」と感じられる価値を提供します。

## ユーザーの獲得について
エンジニア転職用ポートフォリオとして公開

GitHub / Qiita / X（旧Twitter）での発信

「感謝」「日記」「メンタルケア」系の文脈での紹介

まずは小規模に、共感してくれる人に届くことを重視します。

## サービスの差別化ポイント・推しポイント
■差別化ポイント

SNS機能を最小限にし、承認欲求を煽らない設計

「書くこと」「振り返ること」に特化

メンタル回復・自己肯定感にフォーカス

既存の日記アプリや感謝アプリは
機能過多・習慣化が難しいものが多いため、
続けやすさと心理的ハードルの低さを重視します。

■推しポイント

ネガティブな状態でも使える設計

技術だけでなく「思想」が伝わるアプリ

作り手自身の実体験がベースになっている点
